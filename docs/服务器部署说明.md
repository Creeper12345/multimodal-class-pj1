# 服务器部署说明

这份文档面向一台全新的 Linux 服务器，目标是从 0 开始把本项目拉下来、建好环境，并运行 Task 1。

默认前提：

- 服务器是 Linux
- 已安装 NVIDIA 驱动
- CUDA 运行时版本为 12.4
- 你有 conda 或 miniconda

## 1. 登录服务器

```bash
ssh <你的用户名>@<服务器地址>
```

如果你要指定端口：

```bash
ssh -p <端口> <你的用户名>@<服务器地址>
```

## 2. 检查基础环境

先确认 GPU、CUDA、conda、git 都可用。

```bash
nvidia-smi
```

预期：

- 能看到 GPU 型号
- Driver 正常
- CUDA Version 显示 `12.4`

```bash
conda --version
git --version
python --version
```

如果 `conda` 不存在，先安装 Miniconda。

## 3. 拉取项目代码

进入你准备用来放项目的目录：

```bash
mkdir -p ~/projects
cd ~/projects
```

首次拉取：

```bash
git clone https://github.com/Creeper12345/multimodal-class-pj1.git
cd multimodal-class-pj1
```

以后更新代码：

```bash
cd ~/projects/multimodal-class-pj1
git pull origin main
```

查看当前分支和状态：

```bash
git status -sb
```

## 4. 创建 conda 环境

本项目统一使用 `multimodal` 环境。

### 方案 A：直接用脚本

```bash
bash scripts/setup_env.sh multimodal cuda124
```

这个脚本会做几件事：

1. 创建 `multimodal` 环境
2. 安装 CUDA 12.4 对应的 `torch` 和 `torchvision`
3. 安装 `decord`
4. 安装 `requirements.txt`

### 方案 B：手动安装

```bash
conda env create -f environment.yml
conda activate multimodal
python -m pip install --index-url https://download.pytorch.org/whl/cu124 torch==2.5.1 torchvision==0.20.1
python -m pip install decord==0.6.0
python -m pip install -r requirements.txt
```

## 5. 准备数据集

Task 1 默认读取以下路径：

- `datasets/val2017/`
- `datasets/annotations/captions_val2017.json`

所以最终目录结构至少要是这样：

```text
multimodal-class-pj1/
  datasets/
    val2017/
      000000000139.jpg
      ...
    annotations/
      captions_val2017.json
```

如果你已经有 COCO 数据，可以拷贝到仓库目录下：

```bash
mkdir -p datasets
```

例如从别处复制：

```bash
cp -r <你的数据路径>/val2017 datasets/
cp -r <你的数据路径>/annotations datasets/
```

如果你放在别的位置，也可以在运行时手动指定：

```bash
--image-dir /path/to/val2017
--annotation /path/to/captions_val2017.json
```

## 6. 运行前自检

先激活环境：

```bash
conda activate multimodal
```

然后做环境检查：

```bash
python scripts/check_environment.py --check-data
```

预期结果：

- `torch_installed` 为 `true`
- `cuda_available` 为 `true`
- `annotation_exists` 为 `true`
- `image_dir_exists` 为 `true`
- `lavis`、`open_clip`、`decord` import 正常

如果这里失败，不要直接跑正式实验，先把输出发给我。

## 7. 先做 dry-run

```bash
python code/pj1/task1/run_retrieval.py --dry-run --max-images 10
```

预期结果：

- 能显示 `num_images: 10`
- 能显示 `num_captions: 50`
- 最后一行是：

```text
Dry run complete; no model was loaded.
```

## 8. 依次做 smoke test

在 smoke test 之前，建议先统一预下载：

```bash
python scripts/prefetch_task1_models.py
```

如果服务器在国内，建议优先尝试 Hugging Face 镜像：

```bash
python scripts/prefetch_task1_models.py \
  --hf-endpoint https://hf-mirror.com
```

如果服务器访问不到 Hugging Face，需要先在一台能联网的机器上准备好
`bert-base-uncased` tokenizer 目录，再拷贝到服务器，然后执行：

```bash
python scripts/prefetch_task1_models.py \
  --bert-tokenizer-path /path/to/bert-base-uncased
```

项目会把它复制到：

```text
outputs/task1_retrieval/hf_cache/manual/bert-base-uncased/
```

如果只想先下载 BLIP retrieval：

```bash
python scripts/prefetch_task1_models.py \
  --model-spec lavis:blip_retrieval:coco
```

预下载成功后，再跑下面这些命令时加 `--local-files-only`，这样程序只会读取本地缓存，不再联网下载。

### 8.1 CLIP

```bash
python code/pj1/task1/run_retrieval.py \
  --local-files-only \
  --max-images 20 \
  --image-batch-size 4 \
  --text-batch-size 32 \
  --eval-batch-size 128 \
  --model-spec lavis:clip_feature_extractor:base
```

### 8.2 BLIP retrieval

```bash
python code/pj1/task1/run_retrieval.py \
  --local-files-only \
  --bert-tokenizer-path outputs/task1_retrieval/hf_cache/manual/bert-base-uncased \
  --max-images 20 \
  --image-batch-size 4 \
  --text-batch-size 32 \
  --eval-batch-size 128 \
  --model-spec lavis:blip_retrieval:coco
```

### 8.3 BLIP-2

```bash
python code/pj1/task1/run_retrieval.py \
  --local-files-only \
  --bert-tokenizer-path outputs/task1_retrieval/hf_cache/manual/bert-base-uncased \
  --max-images 20 \
  --image-batch-size 2 \
  --text-batch-size 16 \
  --eval-batch-size 128 \
  --model-spec lavis:blip2_feature_extractor:pretrain
```

每次跑完后，结果会出现在：

- `outputs/task1_retrieval/results/`

## 9. 运行完整 Task 1

```bash
python code/pj1/task1/run_retrieval.py \
  --local-files-only \
  --bert-tokenizer-path outputs/task1_retrieval/hf_cache/manual/bert-base-uncased \
  --model-spec lavis:clip_feature_extractor:base \
  --model-spec lavis:blip_retrieval:coco \
  --model-spec lavis:blip2_feature_extractor:pretrain
```

如果显存不够，先减小 batch：

```bash
python code/pj1/task1/run_retrieval.py \
  --local-files-only \
  --bert-tokenizer-path outputs/task1_retrieval/hf_cache/manual/bert-base-uncased \
  --image-batch-size 2 \
  --text-batch-size 16 \
  --eval-batch-size 128 \
  --model-spec lavis:clip_feature_extractor:base \
  --model-spec lavis:blip_retrieval:coco \
  --model-spec lavis:blip2_feature_extractor:pretrain
```
