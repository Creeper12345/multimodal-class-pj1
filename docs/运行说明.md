# 运行说明

这份文档面向迁移到 GitHub 仓库和服务器后的实际执行。

## 1. 运行目标

当前已经完成 Task 1，并开始搭建 Task 2：

- 在 COCO `val2017` 上做 `Text-to-Image Retrieval`
- 在 COCO `val2017` 上做 `Image-to-Text Retrieval`
- 输出 `Recall@1 / Recall@5 / Recall@10`
- 在 COCO `val2017` 上做图像描述生成
- 输出 `BLEU-4 / CIDEr`，并在条件允许时补充 `METEOR / ROUGE-L / SPICE`

主要输出目录：

- `outputs/task1_retrieval/results/`
- `outputs/task1_retrieval/cache/`
- `outputs/task2_captioning/predictions/`
- `outputs/task2_captioning/results/`

## 2. 代码与组件

- `code/pj1/task1/coco.py`
  - 读取 COCO 标注
  - 将图像和 caption 组织成统一的数据结构
- `code/pj1/task1/models.py`
  - 统一封装 LAVIS / Transformers CLIP / OpenCLIP 的 embedding 接口
- `code/pj1/task1/metrics.py`
  - 计算双向检索的 Recall@K
- `code/pj1/task1/run_retrieval.py`
  - Task 1 主入口
- `code/pj1/runtime.py`
  - 统一设置缓存目录、MPS fallback、Tokenizer 环境变量
- `scripts/check_environment.py`
  - 运行前自检
- `scripts/setup_env.sh`
  - 创建设备对应环境
- `scripts/package_submission.py`
  - 打包提交文件
- `code/pj1/task2/coco.py`
  - 读取 Task 2 的 COCO 图像和参考 caption
- `code/pj1/task2/models.py`
  - 统一封装 BLIP / BLIP-2 caption 生成
- `code/pj1/task2/metrics.py`
  - 统一封装 caption 指标评测
- `code/pj1/task2/run_captioning.py`
  - Task 2 主入口
- `scripts/prefetch_task2_models.py`
  - 预下载并校验 Task 2 caption 模型

## 3. 环境安装

### 3.1 Linux 服务器，CUDA 12.4

目标：创建 `multimodal` 环境，并安装适合 CUDA 12.4 的 PyTorch 与项目依赖。

命令：

```bash
bash scripts/setup_env.sh multimodal cuda124
```

预期结果：

- 创建 conda 环境 `multimodal`
- 安装 `torch==2.5.1`、`torchvision==0.20.1`
- 安装 `decord==0.6.0`
- 安装 `requirements.txt` 中的共享依赖

### 3.2 macOS Apple Silicon

目标：在本地进行开发、dry-run 和小规模 smoke test。

命令：

```bash
bash scripts/setup_env.sh multimodal mps
```

预期结果：

- 使用 MPS 版 PyTorch
- 使用 conda-forge 安装 `decord`
- 自动启用 `PYTORCH_ENABLE_MPS_FALLBACK=1`

### 3.3 如果你不想用脚本

Linux CUDA 12.4：

```bash
conda env create -f environment.yml
conda activate multimodal
python -m pip install --index-url https://download.pytorch.org/whl/cu124 torch==2.5.1 torchvision==0.20.1
python -m pip install decord==0.6.0
python -m pip install -r requirements.txt
```

macOS：

```bash
conda env create -f environment.yml
conda activate multimodal
python -m pip install torch==2.5.1 torchvision==0.20.1
conda install -c conda-forge decord -y
python -m pip install -r requirements.txt
```

## 4. 运行前检查

目标：确认 Python、Torch、CUDA/MPS、LAVIS、OpenCLIP、数据路径都可见。

命令：

```bash
conda run -n multimodal python scripts/check_environment.py --check-data
```

预期结果：

- 输出一段 JSON
- `torch_installed` 为 `true`
- 服务器上 `cuda_available` 为 `true`
- 本地 Apple Silicon 上 `mps_available` 可能为 `true`
- `annotation_exists` 和 `image_dir_exists` 为 `true`

## 5. Task 1 运行命令

### 5.0 先把模型统一下载到本地

目标：把 Task 1 需要的模型权重、tokenizer、processor 文件先下载到本地缓存，
后续正式运行时只使用本地文件。

命令：

```bash
conda run -n multimodal python scripts/prefetch_task1_models.py
```

如果你在国内网络环境，建议优先走镜像：

```bash
conda run -n multimodal python scripts/prefetch_task1_models.py \
  --hf-endpoint https://hf-mirror.com
```

如果你只想预下载某一个模型：

```bash
conda run -n multimodal python scripts/prefetch_task1_models.py \
  --model-spec lavis:blip_retrieval:coco
```

如果是国内网络，也可以和镜像参数一起用：

```bash
conda run -n multimodal python scripts/prefetch_task1_models.py \
  --hf-endpoint https://hf-mirror.com \
  --model-spec lavis:blip_retrieval:coco
```

如果服务器无法访问 Hugging Face，但你手里已经有一个本地
`bert-base-uncased` tokenizer 目录，可以这样指定：

```bash
conda run -n multimodal python scripts/prefetch_task1_models.py \
  --bert-tokenizer-path /path/to/bert-base-uncased
```

预期结果：

- 模型会被下载到本地缓存目录
- 生成 `outputs/task1_retrieval/results/prefetch_manifest.json`
- 生成或复用 `outputs/task1_retrieval/hf_cache/manual/bert-base-uncased/`

默认缓存目录：

- `outputs/task1_retrieval/hf_cache/`
- `outputs/task1_retrieval/torch_cache/`

说明：

- 默认会额外做一次最小 encode，确保不只是权重文件到位，连 tokenizer 也已经可用
- 如果你只想下载不做 encode，可以加 `--skip-encode`

### 6.4 指标评测说明

- 默认请求：`Bleu_4`、`CIDEr`
- 可选补充：`METEOR`、`ROUGE_L`、`SPICE`
- 标准 PTB tokenization 依赖 Java
- 如果 Java 不可用，代码会默认写 warning 并退回到简单 tokenization
- `METEOR` 和 `SPICE` 依赖 Java；默认会被跳过，避免 smoke test 卡死
- 若确认 Java 和 SPICE 依赖都已准备好，再设置 `PJ1_ENABLE_JAVA_METRICS=1` 并显式传入 `--metric METEOR` 或 `--metric SPICE`
- 如果要强制报错而不是回退：

```bash
--tokenizer-fallback error --strict-metrics
```

### 6.5 只读取已有 prediction 补充打分

目标：已经生成 caption 后，不重新加载 BLIP / BLIP-2，不重新生成 caption，只读取：

```text
outputs/task2_captioning/predictions/
```

并补充 `METEOR / ROUGE_L / SPICE` 等指标。

本地或无 Java 环境下，先补能直接计算的指标：

```bash
conda run -n multimodal python scripts/evaluate_task2_predictions.py \
  --metric ROUGE_L
```

预期结果：

- 生成 `outputs/task2_captioning/results/extra_metrics_summary.csv`
- 生成 `outputs/task2_captioning/results/<run_name>_extra_metrics.json`

如果服务器尚未安装 Java，先安装：

```bash
conda activate multimodal
conda install -c conda-forge openjdk=11 -y
java -version
```

安装 Java 后，补充 `METEOR` 和 `ROUGE_L`：

```bash
PJ1_ENABLE_JAVA_METRICS=1 python scripts/evaluate_task2_predictions.py \
  --enable-java-metrics \
  --tokenizer-fallback error \
  --metric METEOR \
  --metric ROUGE_L
```

`SPICE` 还需要 Stanford CoreNLP 依赖。若服务器能访问下载源，先执行：

```bash
python -c "from pycocoevalcap.spice.get_stanford_models import get_stanford_models; get_stanford_models()"
```

然后补 `SPICE`：

```bash
PJ1_ENABLE_JAVA_METRICS=1 python scripts/evaluate_task2_predictions.py \
  --enable-java-metrics \
  --tokenizer-fallback error \
  --metric SPICE
```

说明：

- 这个补充脚本不会重新生成 caption。
- `METEOR` 依赖 Java。
- `SPICE` 同时依赖 Java 和 Stanford CoreNLP 模型文件。
- 如果 SPICE 依赖下载失败，不影响主实验的 `Bleu_4 / CIDEr` 结果。
